## 功能设计
建议分成 4 个页面。

# 1. TCP 通信调试页面
功能：

输入 IP 和端口
连接 / 断开设备
发送十六进制指令或字符串指令
接收设备返回数据
显示通信日志
连接失败、超时、断开重连提示


# 2. 串口通信调试页面
功能：

选择 COM 口
设置波特率、数据位、停止位、校验位
打开 / 关闭串口
发送指令
接收返回数据
支持 HEX / ASCII 显示


# 3. Modbus TCP 调试页面
功能：

连接 Modbus TCP Server
读取保持寄存器
写入单个寄存器
显示寄存器地址和值
异常码提示

你可以用现成库，比如：

NModbus



# 4. 通信日志与配置页面
功能：

自动记录发送时间、接收时间、协议类型、发送内容、接收内容、状态
日志导出 Excel
保存连接配置
读取历史配置

## TypeScript Agent 与数据看板方案

项目分成两部分：

1. WinForms 工业通信工具

负责真实通信，包括 TCP、串口、Modbus TCP。

它要做的事：

- 连接设备
- 发送指令
- 接收返回数据
- 记录通信日志
- 保存配置

2. TypeScript Agent 应用

负责读取通信日志、展示数据、分析异常。

它先不要直接控制设备，第一版只做“看数据”和“给建议”。

原因：

工业通信里，乱发一条指令可能导致设备误动作。先让 Agent 分析日志，不直接发指令，更稳。

后续如果要让 Agent 控制发送，也必须加人工确认。

推荐目录：

```text
industrial-comm-tool
├── WinForms 通信工具
└── agent-app
    ├── backend
    └── frontend
```

技术选择：

- 前端：React + Vite + TypeScript
- 图表：ECharts
- 后端：Node.js + Express
- 数据：第一版先用 JSON 文件，后面再考虑 SQLite

## 通信日志数据格式

通信日志不要只保存成一段普通文字。

比如不要只存：

```text
发送：01 03 00 00 00 02，成功
```

应该保存成结构化数据：

```json
{
  "id": "1",
  "time": "2026-04-27 10:30:12",
  "protocol": "TCP",
  "direction": "Send",
  "content": "01 03 00 00 00 02",
  "status": "Success",
  "durationMs": 35,
  "errorType": ""
}
```

字段说明：

- id：日志编号
- time：发送或接收时间
- protocol：协议类型，比如 TCP、Serial、ModbusTCP
- direction：方向，Send 表示发送，Receive 表示接收
- content：发送或接收的内容
- status：结果，比如 Success、Failed、Timeout
- durationMs：响应耗时，单位是毫秒
- errorType：异常类型，比如 Timeout、Disconnected、CrcError、ModbusException

这样设计的好处：

- 表格可以直接显示每条通信记录
- 折线图可以用 durationMs 展示响应耗时
- 柱状图可以统计 Success 和 Failed
- 饼图可以统计不同 errorType 的占比
- Agent 可以根据 protocol、status、errorType 生成排查建议

## ECharts 数据展示设计

第一版数据看板做 4 块：

1. 响应耗时折线图

显示设备每次返回用了多久。

作用：看设备是不是越来越慢。

2. 成功 / 失败柱状图

统计一段时间内成功和失败的次数。

作用：看通信是否稳定。

3. 异常类型饼图

统计超时、断开、CRC 错误、Modbus 异常等问题占比。

作用：快速知道主要问题是什么。

4. 实时通信日志表格

显示每一条发送和接收记录。

作用：方便人工排查。

## 推荐学习顺序

第 1 步：先学 TCP 通信，理解 IP、端口、连接、发送、接收。

第 2 步：做 WinForms TCP 页面，先连接本机模拟服务端。

第 3 步：把每次发送和接收记录成结构化日志。

第 4 步：把日志保存成 JSON 文件。

第 5 步：创建 TypeScript Agent 应用，先读取 JSON。

第 6 步：用 ECharts 画响应耗时、成功失败、异常占比。

第 7 步：让 Agent 根据日志给出异常分析。

第 8 步：再接串口和 Modbus TCP。
